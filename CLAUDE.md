## Package Overview

mSigPlot is an R package for plotting mutational signatures and
mutational spectra including SBS (single base substitution), DBS
(doublet base substitution), and indel catalogs. It creates
publication-quality ggplot2 bar plots (and heatmaps for DBS136) for 10
catalog types.

## Architecture

### Dispatch Pattern

`plot_guess(catalog, ...)` and `plot_guess_pdf(catalog, filename, ...)`
auto-detect the catalog type by row count and dispatch to the
appropriate function. Each catalog type has a paired `plot_*()` (single
sample, returns ggplot) and `plot_*_pdf()` (multi-sample PDF, 5 per page
via
[`gridExtra::grid.arrange`](https://rdrr.io/pkg/gridExtra/man/arrangeGrob.html) +
`cairo_pdf`).

| Rows | Function       | Mutation type                       |
|------|----------------|-------------------------------------|
| 1536 | `plot_SBS1536` | SBS pentanucleotide context         |
| 476  | `plot_476`     | Indel with flanking base context    |
| 192  | `plot_SBS192`  | SBS with transcription strand       |
| 166  | `plot_ID166`   | Indel genic/intergenic              |
| 144  | `plot_DBS144`  | DBS with transcription strand       |
| 136  | `plot_DBS136`  | DBS (heatmap layout, 10 4x4 panels) |
| 96   | `plot_SBS96`   | SBS trinucleotide context           |
| 89   | `plot_89`      | Indel (Koh classification)          |
| 83   | `plot_83`      | Indel (COSMIC ID83)                 |
| 78   | `plot_DBS78`   | DBS                                 |

### Catalog Row Ordering

[`catalog_row_order()`](https://steverozen.github.io/mSigPlot/reference/catalog_row_order.md)
returns a named list of canonical row name vectors for all supported
types (SBS96, SBS192, SBS1536, DBS78, DBS136, DBS144, ID, ID166, ID89,
ID476). Strand-aware types (SBS192, DBS144) have internal
`reorder_*_for_plotting()` helpers that interleave
transcribed/untranscribed pairs. DBS136 uses
[`order_DBS136_for_plotting()`](https://steverozen.github.io/mSigPlot/reference/order_DBS136_for_plotting.md)
to map 136 rows into a 160-position heatmap grid.

### Indel Type Definitions

The 476-channel indel type table is generated by `type_476_indel_type()`
(R/type_476_indel_type.R). The 89-channel table is defined inline within
`plot_89()`. Both produce data frames with columns: IndelType, Indel,
Indel3, Figlabel.

### Shared Conventions

- **Y-axis auto-detection**: If `sum(catalog) < 1.1`, label is
  “Proportion”; otherwise “Count”. Some functions also support a
  `catalog.type` attribute on the catalog object (`"counts"`,
  `"counts.signature"`, `"density"`, `"density.signature"`).
- **`show_counts` parameter**: `NULL` (auto based on catalog type),
  `TRUE` (force), or `FALSE` (suppress). Controls per-class count
  labels.
- **Text size parameters**: Named `*_cex` (multipliers relative to
  `base_size`). Deprecated `*_size` aliases still accepted with
  warnings; specifying both old and new errors.
- **Color palettes**: Hardcoded per function for consistent publication
  figures.
- **PDF export**: All `*_pdf()` functions use
  [`cairo_pdf()`](https://rdrr.io/r/grDevices/cairo.html) and arrange 5
  plots per page (except DBS136 heatmaps: 1 per page).
- **`plot_complex` parameter**: Indel plots (89, 476) have an optional
  Complex channel, off by default.
